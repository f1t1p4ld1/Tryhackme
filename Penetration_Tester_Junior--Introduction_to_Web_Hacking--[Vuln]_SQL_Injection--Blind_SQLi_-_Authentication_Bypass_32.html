Blind SQLi - Authentication Bypass


BLIND SQLi

A diferencia de la inyeccion SQL en banda donde podemos ver los resultados de nuestro ataque directamente en la pantalla, la Blind SQLi ocurre cuando recibimos poca o ninguna retroalimentacion para confirmar si nuestras consultas

inyectadas fueron exitosas. Esto se debe a que los mensajes de error se han desactivado, pero la inyeccion sigue funcionando, quizas le sorprenda que solo necesitamos esa corta retroalimentacion para enumerar correctamente una base de

datos completa

Authentication Bypass

Una de las tecnicas de inyeccion SQL a ciegas mas sencillas es cuando se omiten metodos de autenticacion como formularios de inicio de sesion. En este caso, no estamos tan interesados en recuperar datos de la base de datos,

simplemente pasar el inicio de sesion.

Los formularios de acceso que estan conectados a una base de datos de usuarios a menudo se desarrollan de tal manera que la aplicacion web no este interesada en el contenido del nombre de usuario y la clave, pero mas en si ambos

forman un par coincidente en la tabla de usuarios, en terminos basicos, la aplicacion web pregunta a la base de datos: tiene un usuario con el nombre de usuario bob y la clave bob123? la base de datos responde con si o no(verdadero/

falso) y dependiendo de esa respuesta, se determina si la aplicacion le permite proceder o no.

select * from users where username='%username%' and password='%password%' LIMIT 1;
